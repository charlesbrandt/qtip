Traditional approach

The standard, traditional way to do complex layout with HTML is with floats and different display properties. That or you have to set absolute positioning very explicitly, which is not very dynamic or responsive. Even with all of these tricks, I have a hard time getting vertical alignment to work without doing something ridiculously kludgey.

Horizontal centering

If you don't want to set a fixed width on the inner div you could do something like this:

#outer {
  width: 100%;
  text-align: center;
}

#inner {
  display: inline-block;
}

That makes the inner div into an inline element that can be centered with text-align.

via: http://stackoverflow.com/questions/114543/how-to-center-a-div-in-a-div-horizontally

// Specify the number of columns and set column and gutter widths
@columns: 12;
@column-width: 60;
@gutter-width: 20;

// Remove the definition below for a pixel-based layout
@total-width: 100%;


////////////
// LAYOUT //
////////////

html {
     height: 100%;
}

body {
     height: 100%;
}

// header
header#top {
    margin-bottom: 1em;
}

.launch-column {
    .column(4);
    height: 100%;
}

.launcher {
    //this sets up the vertical layout for each column
    width: 100%;
    display: inline;
    float: left;
    height: 33%;
}

.container {
    //this sets layout properties for each individual launcher cell
    //and handle horizontal centering
    height: 100%;
    width: 100%;
    display: block;
    text-align: center;
}

.icon {
    display: inline-block;
    vertical-align: middle;
}
